# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Модели данных

#### Класс ProductsCatalog
Класс хранит товары, которые можно купить в приложении.

Конструктор класса не принимает параметров.

Поля класса:  
`products: IProduct[]` - поле для хранения массива всех товаров.
`selectedProduct: IProduct | null` - поле для хранения выбранного товара для подробного отображения.

Методы класса:  
`setProducts(products: IProduct[]): void` - метод сохраняет переданный массив товаров.
`getProducts(): IProduct[]` - метод возвращает массив всех товаров, хранящихся в модели.
`getProductById(id: string): IProduct | undefined` - метод находит и возвращает товар по его идентификатору, переданному в параметре. Если товар не найден, возвращает undefined.
`setSelectedProduct(product: IProduct | null): void` - метод сохраняет переданный товар в поле selectedProduct.
`getSelectedProduct(): IProduct | null` - метод возвращает товар, выбранный для подробного отображения. Если товар не выбран, возвращает null.


#### Класс Basket
Класс хранит товары, которые покупатель выбрал для покупки.

Конструктор класса не принимает параметров.

Поля класса:
`products: IProduct[]` - поле для хранения массива товаров, выбранных покупателем для покупки.

Методы класса:
`getProducts(): IProduct[]` - метод возвращает массив товаров, находящихся в корзине.
`addProduct(product: IProduct): void` - метод добавляет товар, переданный в параметре, в массив корзины.
`deleteProduct(id: string): void` - метод удаляет товар по переданному в параметре идентификатору из массива корзины.
`clear():void` - метод очищает массив корзины.
`getTotalPrice(): number` - метод возвращает стоимость товаров в корзине.
`getProductsCount(): number` - метод возвращает количество товаров в корзине.
`hasProductById(id: string): boolean` - метод проверяет наличие товара в корзине по id, полученному в параметре.

#### Класс Buyer
Класс хранит и управляет данными покупателя, включая способ оплаты, контактную информацию и адрес доставки, которые тот должен указать при оформлении заказа.

Конструктор класса не принимает параметров.

Поля класса:
`payment: TPayment` - поле хранит вид оплаты.
`address: string` - поле хранит адресс покупателя.
`phone: string` - поле хранит телефон покупателя.
`email: string` - поле хранит email покупателя.

Методы класса:
`setData(buyer: IBuyer): void` - метод обновляет только те поля, которые переданы в параметрах, не затрагивая остальные.
`getData(): IBuyer` - метод возвращает объект с текущими данными покупателя.
`clear(): void` - метод очищает все данные покупателя, устанавливая их в начальное состояние.
`validate(): { isValid: boolean; errors: { [K in keyof IBuyer]?: string } }` - метод проверяет, что все поля заполнены (не пустые). Возвращает флаг isValid и объект errors, содержащий сообщения об ошибках по каждому полю, если они есть.

### Описание событий 

`catalog:set` - событие генерируется при загрузке массива товаров. Создает карточки товаров в галерее на главной странице.
`card:select` - событие генерируется при нажатии на карточку товара в каталоге. Создает карточку просмотра товара, открывает ее в модальном окне.
`cardPreview:action` - событие генерируется при нажатии на кнопку в карточке просмотра товара. Добавляет/удаляет товар из корзины в зависимости от его наличия в ней.
`basket:open` - событие генерируется при нажатии на иконку корзины в шапке сайта. Открывает модальное окно с содержимым корзины. Создает карточки товаров в корзине.
`basket:change` - событие генерируется при изменении содержимого корзины. Перерисовывает счетчик у корзины.
`card:removeFromBasket` - событие генерируется при удалении товара в корзине. Перерисовывает содержимое корзины.
`modal:close` - событие генерируется при закртии модального окна по кнопке или клику вне окна.
`basket:order` - событие генерируется при нажатии на кнопку оформления заказа.
`buyer:change` - событие генерируется при изменении данных покупателя. Вычисляет ошибки при заполнении формы.
`payment:card` - событие генерируется при выборе метода оплаты онлайн. Обновляет данные покупателя.
`payment:cash` - событие генерируется при выборе метода оплаты наличными. Обновляет данные покупателя.
`address:change` - событие генерируется при вводе адреса. Обновляет данные покупателя.
`email:change` - событие генерируется при вводе почты. Обновляет данные покупателя.
`phone:change` - событие генерируется при вводе телефона. Обновляет данные покупателя.
`order:proceedToContacts` - событие генерируется при нажатии на кнопку продолжения оформления заказа. Открывает модальное окно с формой заполнения почты, телефона и оплаты заказа.
`order:pay` - событие генерируется на нажатии на кнопку оплаты. Отправляет заказ на сервер, открывает модальное окно с сообщением об успешной оплате, товары удаляются из корзины, данные покупателя очищаются.
`orderSuccess:close` - событие генерируется при нажатии на кнопку закрытия модального окна об успешной оплате.

### Презентер
Презентер (Presenter) — это слой приложения, находящийся между моделью данных (Models) и представлением (Views). Его основная задача — координировать логику отображения и обновления данных, а также обрабатывать пользовательские действия, передавая их модели.

В данном приложении презентер реализован в файле main.ts. Это набор подписок на события EventEmitter и логики обработки этих событий, которая напрямую взаимодействует с моделями и представлениями.
